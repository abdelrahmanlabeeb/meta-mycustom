# meta-mycustom

A custom Yocto layer (Scarthgap 5.0) demonstrating a full embedded Linux
driver stack on a `qemuarm64` target: kernel module → character device →
ioctl interface → sysfs attributes → cross-compiled C++ userspace app.

## Dependencies

| Dependency | Branch |
|---|---|
| `poky` (OpenEmbedded-Core) | `scarthgap` |

No additional layers are required.

## Adding the layer to your build

```bash
cd ~/poky
source oe-init-build-env        # drops you into ~/poky/build
bitbake-layers add-layer ../meta-mycustom
```

Or add it manually to `build/conf/bblayers.conf`:

```
BBLAYERS += "/home/<user>/poky/meta-mycustom"
```

## Layer contents

### `recipes-hello/hello`

A minimal "hello world" C application. Cross-compiled with `${CC}`, installed
to `/usr/bin/hello`.

```
IMAGE_INSTALL:append = " hello"
```

### `recipes-kernel/hello-module`

A minimal kernel module demonstrating the init/exit lifecycle and `printk`.
Uses `inherit module`; the Makefile delegates to `$(KERNEL_SRC)`.

```
IMAGE_INSTALL:append = " hello-module"
```

### `recipes-kernel/gpio-sensor`

A simulated GPIO sensor **character device driver**.

- Registers `/dev/gpio-sensor` via `alloc_chrdev_region` + `class_create` +
  `device_create_with_groups` (udev creates the node automatically).
- `read()` returns one 10/8/12-bit ADC sample per call, generated by a
  software LCG (no real hardware needed).
- **ioctl interface** defined in the shared header `gpio-sensor.h`:

  | Command | Direction | Description |
  |---|---|---|
  | `GPIO_SENSOR_RESET` | none | Reset LCG seed and read counter |
  | `GPIO_SENSOR_SET_MODE` | write → kernel | Set ADC resolution (0=normal 10-bit, 1=fast 8-bit, 2=slow 12-bit) |
  | `GPIO_SENSOR_GET_COUNT` | read → userspace | Total `read()` calls since last reset |

- **sysfs attributes** under `/sys/class/gpio_sensor/gpio-sensor/`:

  | File | Access | Description |
  |---|---|---|
  | `mode` | read/write | Current sampling mode (0/1/2) |
  | `count` | read-only | Total readings since last reset |

- The shared header `gpio-sensor.h` is installed to `/usr/include/` so
  userspace recipes can include it with `DEPENDS = "gpio-sensor"`.

```
IMAGE_INSTALL:append = " gpio-sensor"
```

### `recipes-apps/gpio-sensor-app`

A cross-compiled **C++ application** that exercises the full driver stack:

1. Opens `/dev/gpio-sensor` and reads three samples in NORMAL mode.
2. Issues `ioctl GET_COUNT` and cross-checks it against the sysfs `count`
   attribute.
3. Writes `1` (FAST mode) to `/sys/.../mode` and reads a sample — confirms
   the value is ≤ 255.
4. Issues `ioctl SET_MODE` (SLOW mode) and reads a sample — confirms up to
   4095.
5. Issues `ioctl RESET` and verifies `count` returns to 0 via sysfs.
6. Restores NORMAL mode via sysfs write.

```
IMAGE_INSTALL:append = " gpio-sensor-app"
```

## Quick start (QEMU)

```bash
# Build
cd ~/poky
source oe-init-build-env

# Add all packages to build/conf/local.conf:
# IMAGE_INSTALL:append = " hello hello-module gpio-sensor gpio-sensor-app"

bitbake core-image-minimal

# Run
runqemu qemuarm64 nographic
```

Inside QEMU (log in as `root`, no password):

```bash
# Load the driver
modprobe gpio-sensor
dmesg | tail -3         # gpio-sensor: loaded  major=N minor=0

# Verify /dev node and sysfs attrs exist
ls -l /dev/gpio-sensor
ls /sys/class/gpio_sensor/gpio-sensor/

# Quick manual test
cat /dev/gpio-sensor                              # one ADC sample
cat /sys/class/gpio_sensor/gpio-sensor/mode       # 0 (NORMAL)
echo 1 > /sys/class/gpio_sensor/gpio-sensor/mode  # switch to FAST
cat /dev/gpio-sensor                              # should be ≤ 255

# Full demo
gpio-sensor-app
```
